# Lumen Gem Duel

보석 엔진 빌딩과 실시간 순도 미니게임을 결합한 2인 모바일 웹게임 MVP입니다.

## 실행

이 프로젝트는 빌드 도구 없이 바로 동작하는 정적 웹앱입니다.

1. `index.html`을 브라우저에서 엽니다.
2. GitHub에 이 폴더를 업로드합니다.
3. Vercel에서 프로젝트 루트로 이 폴더를 선택해 배포합니다.

## 현재 구현

- 모바일 세로 화면 중심 UI
- 보석 5종: 루비, 사파이어, 에메랄드, 토파즈, 자수정
- 카드 시장 3단계와 카드 구매
- 카드 구매 순간 미니게임 진입
- 미니게임 시작 전 목표, 역할, 점수 룰 브리핑
- 공개 광맥 기반 원석 수집
- 5종 미니게임
  - 루비: 과열 광클
  - 사파이어: 색상 연쇄
  - 에메랄드: 공명 줄다리기
  - 토파즈: 황금 리코셰
  - 자수정: 별빛 리듬
- S/A/B/C/D 순도 등급 판정
- S등급 랜덤 각인
- 방해자 파편 보상
- 후원자 계약
- 명성 15점 승리
- PWA manifest와 서비스 워커
- 실제 생성 이미지 기반 게임 배경과 카드 아트

## 원석 수급 원리

원석 수집은 턴 행동 하나를 소비하는 채굴 액션입니다.

- 매 턴 `이번 광맥`이 공개됩니다.
- 첫 번째 색 원석은 2개, 두 번째와 세 번째 색 원석은 각각 1개를 얻습니다.
- 별가루 광맥은 낮은 확률로 함께 공개됩니다.
- 원석을 수집하면 턴이 넘어가고 다음 광맥이 새로 열립니다.

이 방식은 완전 랜덤 지급보다 직관적입니다. 플레이어는 구매할 카드 비용을 보고, 지금 광맥을 채굴할지 카드 세공을 시작할지 결정할 수 있습니다.

## 추가 미니게임 추천

출시판에서 게임을 더 다양하게 만들 때는 아래 순서가 좋습니다.

1. 루비: 무조준 투척
   목표 지점이 잠깐 보인 뒤 사라지고, 세공자는 드래그 탄도로 원석을 던집니다. 방해자는 바람이나 충돌체를 움직입니다.
2. 사파이어: 수정 기억 카드
   처음 2초 동안 문양 배열을 보여주고 뒤집습니다. 세공자는 지정 문양을 순서대로 찾고, 방해자는 페이크 힌트나 압박 타이머를 만듭니다.
3. 에메랄드: 균형 정원
   흔들리는 결정판을 좌우 입력으로 안정시킵니다. 방해자는 바람 파동을 보내 중심을 흔듭니다.
4. 토파즈: 황금 컵 셔플
   보석이 든 컵이 섞이고 마지막에 정답 컵을 고릅니다. 방해자는 가짜 컵에 반짝임을 넣어 헷갈리게 합니다.
5. 자수정: 예언 궤도
   현재 위치가 아니라 미래 위치를 예측해 별을 맞힙니다. 방해자는 궤도를 왜곡하거나 먼저 선점합니다.
6. 별가루: 프리즘 절단
   무색 원석을 여러 조각으로 자르는 공간 퍼즐입니다. 절단선이 정확할수록 와일드 자원의 가치가 높아집니다.

정식 출시에서는 보석 5종마다 2개씩, 별가루 1~2개를 더해 총 11~12개 정도가 적당합니다. 한 판에서 같은 미니게임이 너무 자주 반복되지 않도록 최근 2회 등장한 게임은 가중치를 낮추는 방식이 좋습니다.

## 출시 확장 방향

현재는 한 기기에서 번갈아 조작하는 로컬 2인 시연 구조입니다. 정식 출시를 목표로 할 때는 아래 순서가 좋습니다.

1. Firebase Realtime Database, Supabase Realtime, Colyseus 중 하나로 방 코드 동기화를 붙입니다.
2. `state` 변경 지점을 네트워크 액션으로 감쌉니다.
3. 미니게임 입력만 고빈도 이벤트로 보내고, 결과 판정은 방장 또는 서버에서 확정합니다.
4. Capacitor나 Expo WebView로 앱 패키징을 진행합니다.
5. 모바일 터치 테스트 후 미니게임별 시간과 점수표를 조정합니다.

## Vercel 설정

별도 빌드 명령은 필요 없습니다.

- Framework Preset: Other
- Build Command: 비워두기
- Output Directory: `.`

루트에 `vercel.json`이 있어 이미지 캐시 헤더가 적용됩니다.
